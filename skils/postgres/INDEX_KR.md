# INDEX_KR

## 한 줄 설명
PostgreSQL 데이터베이스에 대해 안전한 읽기 전용 조회를 수행하는 스킬입니다.

## 언제 쓰면 되나
- DB 안 데이터를 직접 확인하고 싶을 때
- 테이블 목록, 스키마 구조, 샘플 데이터를 살펴보고 싶을 때
- 분석용 `SELECT` 쿼리를 실행하고 싶을 때
- 운영 DB를 건드리지 않고 안전하게 조회만 해야 할 때

## 이런 상황에 특히 적합
- "현재 users 테이블에 어떤 컬럼이 있지?"
- "최근 주문 20건만 조회해보자"
- 여러 Postgres DB 중 목적에 맞는 DB를 골라 탐색할 때

## 이럴 때는 다른 폴더가 더 맞음
- 쿼리 튜닝과 성능 개선 기준이 필요하면 `supabase-postgres-best-practices`
- 스키마 자체를 설계하는 구조 원칙은 `ddd`도 참고 가능

## 빠른 예시
- 읽기 전용으로 프로덕션 DB 상태 확인
- 테이블 구조 조사
- 특정 조건의 데이터 샘플 조회

## 메모
- 쓰기 쿼리는 막혀 있으므로 비교적 안전합니다.
- 접속 정보 설정과 권한 관리가 먼저 필요합니다.
