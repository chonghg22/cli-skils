# INDEX_KR

## 한 줄 설명
이미 작성된 구현 계획서를 순서대로 읽고 실행하는 스킬입니다.

## 언제 쓰면 되나
- 설계와 계획은 끝났고, 이제 정해진 작업을 차례대로 구현해야 할 때
- 별도 세션이나 독립된 흐름에서 계획 기반으로 개발을 진행하고 싶을 때
- 각 단계마다 검증을 놓치지 않고 진행하고 싶을 때

## 이런 상황에 특히 적합
- 구현 계획 문서가 이미 있는 프로젝트
- 작업 도중 임의 추정보다 "계획서대로 실행"이 중요한 경우
- 중간에 막히면 멈추고 검토해야 하는 보수적 개발 흐름

## 이럴 때는 다른 폴더가 더 맞음
- 아직 설계가 없다면 `brainstorming`
- 같은 세션에서 독립 작업을 병렬 처리하고 싶으면 `subagent-driven-development`

## 빠른 예시
- 승인된 구현 계획서를 바탕으로 기능 개발 시작
- 세부 태스크가 정리된 문서를 따라 순차 구현

## 메모
