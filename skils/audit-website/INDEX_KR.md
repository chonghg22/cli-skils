# INDEX_KR

## 한 줄 설명
웹사이트를 SEO, 성능, 접근성, 보안, 콘텐츠 품질 관점에서 자동 진단하는 스킬입니다.

## 언제 쓰면 되나
- 운영 중인 웹사이트나 웹앱의 전체 건강 상태를 빠르게 점검하고 싶을 때
- 메타 태그, 구조화 데이터, broken link, 모바일 대응, 보안 헤더 같은 문제를 찾고 싶을 때
- 수정 전후 점수를 비교하면서 개선 효과를 확인하고 싶을 때
- 릴리스 전 사이트 품질 점검 리포트를 만들고 싶을 때

## 이런 상황에 특히 적합
- "왜 검색 유입이 안 나오지?" 같은 SEO 점검
- "배포 후 성능이나 접근성 문제가 생겼나?" 확인
- "보안 헤더나 노출된 시크릿이 있나?" 검사

## 이럴 때는 다른 폴더가 더 맞음
- 코드 구현 전에 아이디어를 정리하고 싶으면 `brainstorming`
- 실제 코드 수정 작업을 계획대로 병렬 수행하고 싶으면 `subagent-driven-development`

## 빠른 예시
- 새 마케팅 사이트 배포 후 전체 품질 점검
- 리뉴얼 전후 사이트 점수 비교
- 고객사 홈페이지 SEO/접근성 감사 리포트 생성

## 메모
- 이 스킬은 `squirrel` CLI 설치가 필요합니다.
- 가능하면 로컬보다 실제 배포 URL 기준으로 점검하는 편이 정확합니다.
