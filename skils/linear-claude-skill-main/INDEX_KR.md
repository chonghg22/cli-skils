# INDEX_KR

## 한 줄 설명
Linear 이슈, 프로젝트, 팀 워크플로를 비교적 본격적으로 관리하는 스킬입니다.

## 언제 쓰면 되나
- Linear에서 이슈 생성, 상태 변경, 댓글 추가, 프로젝트 연결을 해야 할 때
- 단순 이슈 조회를 넘어서 프로젝트 단위로 작업 흐름을 운영할 때
- 이슈 설명, 라벨, 프로젝트 배정을 체계적으로 맞추고 싶을 때
- 팀이 Linear를 실제 업무 관리 도구로 사용 중일 때

## 이런 상황에 특히 적합
- 새 기능 작업을 프로젝트와 하위 이슈까지 세팅
- 이슈 생성 시 설명/라벨/프로젝트를 빠짐없이 붙여야 하는 팀 규칙이 있을 때
- 배치로 상태를 업데이트하거나 진행 상황을 정리할 때

## 이럴 때는 다른 폴더가 더 맞음
- 더 가볍게 Linear CLI만 쓰고 싶으면 `linear-cli-skill-main`
- 구현 전에 요구사항 설계부터 필요하면 `brainstorming`

## 빠른 예시
- 신규 프로젝트 생성 후 관련 이슈 일괄 등록
- `ENG-123` 상태를 `Done`으로 바꾸고 코멘트 남기기
- 스프린트/페이즈 단위로 이슈를 올바른 프로젝트에 배치

## 메모
- API 키와 환경변수 보안 취급이 중요합니다.
- 단순 조회보다 "운영 규칙 포함한 관리"에 더 강한 스킬입니다.
