# INDEX_KR

## 한 줄 설명
구현 계획을 독립 태스크로 쪼개고, 서브에이전트를 나눠 투입해 병렬 개발하는 스킬입니다.

## 언제 쓰면 되나
- 구현 계획은 이미 있고, 태스크들이 서로 비교적 독립적일 때
- 같은 세션에서 여러 작업을 빠르게 병렬 처리하고 싶을 때
- 각 태스크마다 구현 후 스펙 리뷰와 코드 품질 리뷰를 따로 거치고 싶을 때

## 이런 상황에 특히 적합
- 파일/모듈별로 작업 영역이 분리되는 기능 개발
- 한 사람이 순차로 하기엔 오래 걸리는 다중 태스크 구현
- 구현 품질을 높이기 위해 "구현 -> 스펙검토 -> 품질검토" 루프를 돌리고 싶을 때

## 이럴 때는 다른 폴더가 더 맞음
- 아직 계획 문서가 없으면 `brainstorming`
- 병렬화보다 순차 실행이 더 안전하면 `executing-plans`

## 빠른 예시
- API, UI, 테스트 작업을 나눠 병렬 처리
- 여러 파일에 흩어진 리팩터링 태스크를 분리 실행

## 메모
- 태스크들이 서로 강하게 얽혀 있으면 오히려 비효율적일 수 있습니다.
- 계획 문서 품질이 높을수록 효과가 큽니다.
