# INDEX_KR

## 한 줄 설명
구현 전에 요구사항, 방향, 설계를 대화로 정리하는 브레인스토밍 스킬입니다.

## 언제 쓰면 되나
- 새 기능을 만들기 전에 무엇을 어떻게 만들지 먼저 정리하고 싶을 때
- 요청이 애매해서 바로 코딩하면 방향이 틀릴 위험이 있을 때
- 사용자 의도, 제약, 성공 기준을 먼저 분명히 해야 할 때
- 구현 접근법을 2~3개 비교하고 설계를 확정하고 싶을 때

## 이런 상황에 특히 적합
- "대시보드 하나 만들자"처럼 범위는 있는데 세부 요구가 비어 있을 때
- 기능은 단순해 보여도 UX, 데이터 흐름, 예외 처리를 먼저 잡아야 할 때
- 팀 내에서 설계 문서를 남기고 승인받은 뒤 개발하고 싶을 때

## 이럴 때는 다른 폴더가 더 맞음
- 이미 구현 계획서가 있고 바로 실행만 하면 되면 `executing-plans`
- 같은 세션에서 여러 독립 작업을 병렬로 진행하려면 `subagent-driven-development`

## 빠른 예시
- 신규 결제 플로우 설계
- 관리자 페이지 구조 논의
- 기존 기능 개편 전 요구사항 정리와 설계 문서 작성

## 메모
- 이 스킬의 핵심은 "바로 구현하지 않고 먼저 설계 승인을 받는 것"입니다.
