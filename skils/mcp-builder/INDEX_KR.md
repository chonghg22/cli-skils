# INDEX_KR

## 한 줄 설명
외부 API나 서비스를 LLM이 다룰 수 있도록 MCP 서버를 설계하고 구현할 때 쓰는 가이드 스킬입니다.

## 언제 쓰면 되나
- 새 MCP 서버를 만들고 싶을 때
- 단순 API 래핑이 아니라, 에이전트가 쓰기 좋은 도구 형태로 설계해야 할 때
- Python 또는 Node/TypeScript로 MCP 서버를 구현하려고 할 때
- 입력/출력 포맷, 에러 메시지, 평가 기준까지 포함해 품질 있게 만들고 싶을 때

## 이런 상황에 특히 적합
- 사내 API를 MCP로 연결해 LLM이 쓰게 만들기
- Notion, Jira, 내부 플랫폼 같은 서비스를 MCP 서버로 노출
- 도구 설계 자체를 "API 중심"이 아니라 "작업 흐름 중심"으로 재구성해야 할 때

## 이럴 때는 다른 폴더가 더 맞음
- 이미 있는 MCP 서버를 그냥 쓰기만 하면 되면 이 스킬까지는 과할 수 있음
- Postgres 조회만 필요하면 `postgres`

## 빠른 예시
- 사내 티켓 시스템용 MCP 서버 제작
- 캘린더/CRM/결제 시스템 연동용 에이전트 도구 설계

## 메모
- 이 스킬은 구현보다 앞단의 리서치와 도구 설계 원칙이 매우 중요합니다.
