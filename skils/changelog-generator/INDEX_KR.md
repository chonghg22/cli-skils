# INDEX_KR

## 한 줄 설명
Git 커밋 기록을 읽어 사용자 친화적인 변경사항 문서로 바꿔주는 스킬입니다.

## 언제 쓰면 되나
- 릴리스 노트를 써야 할 때
- 지난 일주일이나 한 달간 제품 업데이트를 요약해야 할 때
- 기술적인 커밋 메시지를 고객용 문장으로 바꾸고 싶을 때
- 공개용 `CHANGELOG.md`를 유지하고 싶을 때

## 이런 상황에 특히 적합
- 버전 `v1.8.0` 배포 전에 변경사항 정리
- 앱스토어 업데이트 설명 작성
- 사내 공유용 배포 내역 문서 작성

## 이럴 때는 다른 폴더가 더 맞음
- 코드 품질 규칙 자체를 개선하려면 `ddd`
- 작업 계획을 실행하려면 `executing-plans`

## 빠른 예시
- "지난 릴리스 이후 커밋으로 changelog 만들어줘"
- "이번 주 사용자에게 보여줄 업데이트 요약 정리해줘"

## 메모
- 초안 자동화에는 매우 좋지만, 최종 공개 전에는 사람이 한번 다듬는 편이 좋습니다.
