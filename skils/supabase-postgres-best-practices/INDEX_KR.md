# INDEX_KR

## 한 줄 설명
Supabase가 정리한 Postgres 성능 최적화와 운영 모범 사례 모음입니다.

## 언제 쓰면 되나
- SQL 쿼리가 느린 이유를 찾고 싶을 때
- 인덱스, 스키마, RLS, 커넥션 관리 기준이 필요할 때
- Postgres 리뷰를 할 때 "뭘 봐야 하는지" 체크리스트가 필요할 때
- Supabase 또는 일반 Postgres 환경에서 성능 문제를 줄이고 싶을 때

## 이런 상황에 특히 적합
- 누락된 인덱스 때문에 조회가 느린 경우
- RLS 정책 때문에 성능 저하가 있는 경우
- connection pooling, vacuum/analyze, query plan 같은 운영 이슈 검토

## 이럴 때는 다른 폴더가 더 맞음
- 실제 DB 데이터 조회만 하면 되면 `postgres`
- 전체 코드 구조와 도메인 설계는 `ddd`

## 빠른 예시
- 느린 쿼리 리뷰
- 테이블 설계와 인덱스 전략 점검
- Supabase 프로젝트의 DB 성능 가이드 참고

## 메모
- 이 폴더는 "실행 도구"라기보다 "레퍼런스와 기준서"에 가깝습니다.
