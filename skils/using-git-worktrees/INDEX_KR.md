# INDEX_KR

## 한 줄 설명
현재 작업 공간을 건드리지 않고 기능 개발용 분리 작업 디렉토리를 만드는 Git worktree 스킬입니다.

## 언제 쓰면 되나
- 새 기능 작업을 기존 워크스페이스와 분리해서 시작하고 싶을 때
- 동시에 여러 브랜치를 다뤄야 할 때
- 구현 계획 실행 전에 안전한 독립 작업 공간이 필요할 때
- 현재 작업 디렉토리를 유지한 채 다른 작업을 병행해야 할 때

## 이런 상황에 특히 적합
- 메인 작업과 핫픽스를 동시에 진행
- 대규모 기능을 별도 worktree에서 시작
- 병렬 에이전트 작업 전에 깨끗한 기준 상태 확보

## 이럴 때는 다른 폴더가 더 맞음
- 단순 파일 정리는 `file-organizer`
- 구현 작업 병렬화는 `subagent-driven-development`

## 빠른 예시
- `feature/auth`용 별도 worktree 생성
- 테스트 통과하는 깨끗한 기준 브랜치에서 새 작업 시작

## 메모
- `.worktrees` 같은 디렉토리가 Git에 추적되지 않도록 확인하는 절차가 중요합니다.
- 보통 구현 시작 전 준비 단계에서 먼저 쓰는 스킬입니다.
